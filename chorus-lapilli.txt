Chorus Lapilli Implementation:

First thing I decided to tackle was the difference between traditional tic-tac-toe and this version, the switch between placing pieces and moving them(after both players have moved 3 times). I started by adding a function to board that finds the number of Xs and Os on the board, and returns handleClick early if that number for whichever one is up next exceeds 3(effectively stopping placement).

Noticed a duplicate if statement call(using xIsNext twice to decide whether character was an X or O), so I refactored the rest of the function to prevent any duplicate if statements(if statement already needed to be called before for player). At this point, on the user side it looks as expected, where no more moves were possible after 3 X's were put down. 

Then I started to tackle the moving portion, where the first click has to select and the second has to place it. Started with implementing a new var startSquare(via useState, board needs to remember it), and fed it into handleClick. Then, I started the moving portion, where I refactored my conditional so that it first handled the placement phase, then started handling the movement phase.

For the movement phase, I started by handling the startSquare portion first(startSquare is null), with an extra check to make sure the player selected a square that was their own. Then, I handled the destination Square portion, which checked whether the new square was empty and adjacent to the startSquare(using a helper function, otherwise readability would have suffered), and updates the squares if both checks were satisfied, then resets startSquare.
